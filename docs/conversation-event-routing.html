ABC Canvas 会話イベントの流れ

目的は、GUIの会話タブとDiscordに「同じ意味の状態」が見えるようにすることです。実装の内部ログをそのまま出すのではなく、人間が読める会話イベントに変換して流します。

基本ルール: 会話に出すものは、必ずひとつの出口 emit_conversation_event を通します。

この出口が、DB保存、GUI表示、Discord送信、サーバーログをまとめて扱います。保存した場所ごとにDiscord送信を後付けしません。

なぜ変えるのか

今は、会話DBへの保存場所が複数あり、それぞれが個別にDiscordへ送ったり送らなかったりしています。そのため、親タブにはあるのにDiscordには出ない、途中経過は出るが巨大なJSONになる、というズレが起きています。

直すべきなのは、漏れた場所へ送信処理を足すことではありません。すべての「人間に見せる会話イベント」を同じ出口へ集めることです。

一本化した流れ

1. 発生ユーザー発言、ポワン返答、子ポワン報告、Codex途中経過などが発生する。
2. 整形内部コマンドやJSONを、人間向けの短い文章へ変換する。
3. 保存DBの会話メッセージとして保存する。
4. 配信GUIで見え、必要なものだけDiscordにも流れる。

会話イベントの種類

kind 意味 GUI Discord
operator_message ユーザー本人の発言。 出す 原則出さない。Discord発言はDiscord側に既にあるため。
assistant_reply ポワンの最終返答。 出す 出す
codex_progress 作業途中。コマンド開始、完了、ファイル変更など。 短く出す 短く出す
child_command_sent 親ポワンが子ポワンへ指示を送った事実。 親タブと子タブに出す 対象ポワンなら出す
child_report_returned 子ポワンが親ポワンへ報告した内容。 親タブと子タブに出す 対象ポワンなら出す
internal_log 内部診断用ログ。巨大JSON、PID終了行、低レベルstdoutなど。 出さない 出さない

人間に見せる文章の基準

見せる文章は「今なにが起こったか」を一目で分かる形にします。内部コマンド、長いJSON、token数だけの行は、そのまま会話へ出しません。

悪い表示

コマンド完了 exit=0: "C:\Windows\System32\WindowsPowerShell\v1.0\powershell.exe" -Command "@' { "instruction": "...", "instructions": [...巨大JSON...] }" 出力: { "ok": true, "result": { "status": "accepted", "sentChildren": [...], "skippedChildren": [...] }}

良い表示

子ポワン1件へ指示を送信しました。 送信先: 今だけの返事をする 対象外: 6件 状態: accepted

親子ポワンの例

親から子への指示

親タブにも子タブにも、同じ出来事が分かる形で残します。ただし、子Codexへ実際に渡す本文と、人間向けに親タブへ見せる本文は分けます。

X投稿収集ポワン から 今だけの返事をする へ指示を送りました。 指示: 短く、返事が返せることだけを返してください。 状態: accepted

子から親への報告

子報告は親タブにもDiscordにも見えるべきです。親ポワンが後で要約した返事だけでは足りません。

今だけの返事をする から報告: 返事、返せるよ。 状態: completed

実装方針

STORE.append_conversation_message... を直接呼ぶ場所を、順番に emit_conversation_event へ置き換えます。まずは新しく問題になっている経路から移行し、最終的に個別のDiscord送信関数を減らします。

この設計なら、「親タブにはあるがDiscordにない」「Discordだけ出る」「巨大JSONが流れる」というズレを減らせます。
