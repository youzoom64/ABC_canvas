# ABC Canvas

ABC Canvas は、思いついたアイデアを小さな意味に分けて、最後にコードや設計へ近づけるためのローカルWebアプリです。

たとえば「ログ機能を作りたい」と思ったら、まず大きなアイデアを1つ置きます。そこから「ログレベル」「ファイル出力」「例外記録」のように子要素へ分けます。さらに必要なら、その子要素をもっと小さく分けます。

この1つ1つの意味のかたまりを、このアプリでは「ポワン」と呼びます。

## まず動かす

```powershell
.\start_abc_canvas.bat
```

ブラウザで開きます。

```text
http://127.0.0.1:8790/
```

手動で起動する場合:

```powershell
python -m pip install -r requirements.txt
python -m uvicorn app:app --host 127.0.0.1 --port 8790 --no-access-log
```

## ポワンに作らせる流れ

ABC Canvas は、手で全部の箱を並べるだけのツールではありません。

まず親になるポワンへ「何を作りたいか」を話します。ポワンはその意味を受け取り、必要な子ポワンを考えます。子ポワンができたら、それぞれの子ポワンにも自分の意味を考えさせ、さらに必要なら孫ポワンを作らせます。

流れ:

1. 最初のポワンに、作りたいものを伝える
2. そのポワンに名前と意味を決めさせる
3. 必要な子ポワンを作らせる
4. 子ポワンごとに役割を考えさせる
5. まだ大きい子ポワンは、さらに孫ポワンへ分けさせる
6. これ以上分けないポワンにコードを書かせる
7. 親ポワンに、子ポワン同士のつなぎ方を考えさせる

例:

```text
親ポワン:
  Python共通ログ関数を作りたい

子ポワン:
  ログレベル
  ロガー取得
  コンソール出力
  ファイル出力
  例外記録

最終ポワン:
  TRACEレベル定義
  ファイルハンドラ生成
  例外メッセージ整形
```

この流れでは、ユーザーは最初から全部を設計しません。まず大きな意味を置き、ポワンに分解させ、必要なところだけ人間が直します。

## 手で操作する流れ

1. プロジェクトを作る
2. 最初のポワンに、作りたいものを書く
3. `Add Child` で子ポワンを作る
4. 子ポワンに役割を書く
5. 必要なら子ポワンをさらに分ける
6. 実装できる粒度になったらコードを書く
7. `Save` で保存する

例:

```text
Python共通ログ関数
  ログレベル
  ロガー取得
  コンソール出力
  ファイル出力
  例外記録
```

## ポワンの種類

ポワンには2種類あります。

```text
nerve  まだ分解できる意味
organ  これ以上分けず、コードを書く意味
```

迷ったら、子を持つものは `nerve`、コードを書く最終単位は `organ` にします。

## 画面でできること

- 左のツリーで全体構造を見る
- 中央のキャンバスでポワンを動かす
- 右のパネルで名前、説明、種類、見た目を編集する
- ポワンをダブルクリックして、その内側の世界へ入る
- 右クリックから会話、コード編集、整列、コピー、削除をする

## 保存される場所

プロジェクトはここに保存されます。

```text
powan_work/<project_name>/project.powan
```

会話履歴や作業ログもローカルに保存されます。

## AIと話す

ポワンごとに会話できます。

会話すると、AIには次の情報が渡されます。

- 今のポワン
- 親ポワン
- 子ポワン
- 入力した文章
- 必要に応じてコードや添付

つまり、ただのチャットではなく「この意味について相談する」ための会話です。

## API / CLI

ブラウザ以外からもポワンを操作できます。

```powershell
python abc_powan_tool.py --help
```

よく使うAPI:

```text
GET  /api/projects
POST /api/projects
GET  /api/doc/{name}
POST /api/doc/{name}
GET  /api/ai/powans
POST /api/ai/powans
PATCH /api/ai/powans/{node_id}
POST /api/ai/powans/{node_id}/children
POST /api/ai/powans/{node_id}/actions/tree
```
