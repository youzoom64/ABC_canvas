Python共通ログ関数

複数のPythonアプリケーションで同じ感覚で使える、標準 logging の入口です。 コンソール表示、ファイル保存、ログレベル、例外記録、アプリ別設定をひとつの呼び口にまとめます。

get_logger configure_logging TRACE対応 ファイル出力 例外記録 環境変数設定

最短の使い方

ふだんは get_logger("アプリ名") を呼ぶだけで始められます。 車のエンジン始動ボタンみたいに、まずここを押せばログが走り出します。 

from common_logging import get_logger

logger = get_logger("my_app")

logger.info("起動しました")
logger.warning("注意が必要です")
logger.error("処理に失敗しました")

ファイルにも保存する

file=True を渡すと、アプリ名ごとのログファイルを作ります。 保存先は log_dir で指定できます。 

from common_logging import get_logger

logger = get_logger(
    "my_app",
    level="DEBUG",
    file=True,
    log_dir="logs",
)

logger.debug("開発中の細かい情報")
logger.info("通常の記録")

ファイル名を指定しない場合は、アプリ名から安全なファイル名を作って logs/my_app.log のように保存します。

細かく設定する

設定内容や作られたハンドラまで見たい時は configure_logging を使います。 これは整備モードです。どういうログ配線になったかを確認できます。 

from common_logging import configure_logging

setup = configure_logging(
    "batch_worker",
    app={
        "level": "INFO",
        "console": True,
        "file": True,
        "log_dir": "logs",
        "rotation": "size",
        "max_bytes": 1_000_000,
        "backup_count": 3,
    },
)

logger = setup.logger
logger.info("設定済みロガーで開始")

print(setup.profile.as_dict())

例外を記録する

例外は log_exception に渡すと、例外名、メッセージ、スタックトレース、 任意の文脈情報をまとめて残せます。事故記録のブラックボックスです。 

from common_logging import get_logger, log_exception

logger = get_logger("my_app", file=True)

try:
    1 / 0
except Exception as exc:
    log_exception(
        logger,
        "計算に失敗しました",
        exc,
        context={
            "job_id": "daily-001",
            "user_id": 123,
            "api_token": "secret-value",
        },
    )

password、token、secret、api_key などを含むキーは *** にマスクします。

ログレベル

6段階のログレベルを扱います。TRACE は DEBUG より細かい追跡用です。

レベル 意味 使う場面
TRACE かなり細かい実行追跡 関数の入口、分岐、細部の流れを追う
DEBUG 開発中の状態確認 変数、設定、処理途中の値を見る
INFO 通常の記録 起動、完了、主要な処理の通過を残す
WARNING 注意が必要 処理は続けられるが、放置すると危ない
ERROR 処理失敗 一部処理が失敗したが、アプリ全体は続く可能性がある
CRITICAL 致命的異常 アプリ継続が危ない、すぐ見るべき状態

設定キー

キー 例 説明
level "DEBUG" 出力する最小ログレベルです。
console True コンソールへ表示するかを指定します。
file True ログファイルへ保存するかを指定します。
log_dir "logs" ログファイルの保存先ディレクトリです。
filename "app.log" 保存ファイル名です。省略時はアプリ名から作ります。
format "%(asctime)s %(levelname)s %(message)s" 標準 logging.Formatter の書式です。
rotation "none" / "size" / "time" ログローテーション方式です。
max_bytes 1000000 rotation="size" の最大サイズです。
backup_count 3 ローテーション時に残す世代数です。
propagate False 親ロガーへ伝播するかを指定します。

環境変数で上書きする

LOG_ で始まる環境変数を使うと、実行環境ごとに設定を差し替えられます。 

$env:LOG_LEVEL = "DEBUG"
$env:LOG_FILE = "true"
$env:LOG_DIR = "logs"
$env:LOG_ROTATION = "size"
$env:LOG_MAX_BYTES = "1000000"
$env:LOG_BACKUP_COUNT = "3"
from common_logging import configure_logging

setup = configure_logging("my_app")
setup.logger.debug("環境変数でDEBUGが有効になります")

JSONまたはTOML設定ファイル

アプリ別の設定ファイルを渡すこともできます。JSONはそのまま使えます。 TOMLはPython 3.11以上の tomllib が必要です。 

{
  "level": "INFO",
  "console": true,
  "file": true,
  "log_dir": "logs",
  "filename": "my_app.log"
}
from common_logging import configure_logging

setup = configure_logging("my_app", app="logging.json")
setup.logger.info("設定ファイルから起動しました")

おすすめの配置

複数アプリで使い回すなら、共通モジュールとして common_logging.py のような名前で置くのが分かりやすいです。 

your_project/
  common_logging.py
  app_a.py
  app_b.py
  logs/

まずは get_logger("アプリ名") から使い始めるのがおすすめです。 必要になったらファイル保存、ローテーション、環境変数上書きを足していくと、無理なく育てられます。
