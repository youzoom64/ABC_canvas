# A2: 座標モデルの一本化（作業目標）

## 目的

「親ポワンを動かすと子が飛ぶ」「ワールド遷移で親子の位置が暴れる」を根から消す。
原因は位置の持ち方が二重で、それを手作業で同期しているため。A2では位置の真実を1つにする。

A1（子の配置をフロント一本化、コミット `03cfdf5`）は完了済みでスコープ外。A2はその続き。

## 現状の原因（A1後も残存）

1. 位置が二重表現になっている。各ノードは絶対座標 `layout` と、親ごとのローカル座標 `nestedLayoutByParent[parentId]` の両方を持つ。
2. 子の絶対 `layout` は「計算した時点の親の位置」に固定される。親が後で動いても子は追従しない。表示は `displayLayoutForNode`（[state.js](../static/js/state.js)）が親原点を引いて相対化するため、親が動くとズレ幅ぶん子が飛ぶ。
3. `enterMeaningInterior`（[render.js](../static/js/render.js)）が「子が1つでも nested 座標を持たない」と `force=true` で全子を再パックし、手で並べた配置を壊す。
4. ワールド遷移のカメラ復元が非対称。`rememberViewportBeforeInterior` で覚えた視点を `worldLeave`（[state_transition.js](../static/js/state_transition.js)）で restore せず捨てている。

## やること

- 位置の真実を `nestedLayoutByParent`（親ローカル）に一本化し、絶対 `layout` は表示時に「親位置 + ローカル」で都度導出する（保存される真実から外す）。
- `layout` を書き込む経路を1関数に集約する。散在する `setNodeLayout` 直書きと `syncChildCoordinatesFromWorld/Nested` の変換を畳む。
- `force` 再パックの条件を厳格化し、既存配置を壊さない。
- ワールド遷移のカメラ復元を対称化する。

## 決定事項（合意済み）

- 子の位置の真実は1つ。親の内部世界（`INTERIOR_STAGE` 座標）における位置で持つ。
- 親の中に見える子チップは、その内部世界座標を親の箱へ**等比縮小**して描く（入った世界と完全一致、余白は許容）。
- `nestedLayoutByParent` は廃止し、チップ位置は世界座標から導出する。チップのドラッグは逆変換で世界座標へ書き戻す。

## 実装ステップ（壊さないため分割）

1. 座標変換の数式を `placement.js` に実装し node テストで検証（等比縮小・逆変換の往復一致）。← 今ここ
2. 入れ子描画・ドラッグ・リサイズ・遷移ターゲットを、新変換へ一斉に差し替え、`nestedLayoutByParent` を削除（入れ子サブシステムは原子的に切替）。
3. 子の保存を親相対に変更し、親移動で子が自動追従するようにする。読み手の絶対前提を導出に置換。
4. `force` 再パック厳格化・カメラ復元対称化。

各ステップ後に grep 証明と実機確認（ユーザー）を挟む。

## 完了の定義（done）

- 位置の真実の表現が1つだけ（grep で確認）。
- `layout` への書き込みが1経路だけ（grep で確認）。
- 配置計算が1か所のまま（A1の状態を維持）。
- 3経路（描画・ドラッグ確定・ワールド遷移）で同じノードの位置が一致する。
- 親を動かすと子が追従する。ワールド遷移の往復で視点が元に戻る。
